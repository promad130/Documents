A shell is a computer program that **acts as an** interface between the user and the operating system's kernel. 
Kernel is what makes things happen in a computer hardware. It handles everything.

**Its primary role** is to interpret and execute the commands entered by the user, either interactively (one command at a time) or through scripts (a series of commands in a file).

# It is:
- **Command Interpreter:** The shell reads commands typed by the user and translates them into actions that the operating system can perform. It serves as a command-line interpreter, processing each command and providing feedback or results
- **Types of Shells:** There are different types of shells, including command-line shells (like Bash, Zsh, and Csh) and graphical shells (which provide a GUI for interacting with the system)
- **Location in the OS:** The shell operates in user space, acting as the intermediary between the user and the kernel (the core part of the operating system that manages hardware and system resources)
- **Shell Scripting:** Shells support scripting, allowing users to automate tasks by writing scripts—text files containing sequences of commands executed by the shell
- **Examples:** Common shells include [[Bash]] (Bourne Again Shell), Zsh, Csh, and Ksh on Unix-like systems, and Command Prompt or PowerShell on Windows
- **Not Just CLI:** While most shells are command-line based, some graphical user interfaces (GUIs) are also considered shells because they provide a way for users to interact with the kernel and system services.
