## 3.1 What Is Machine Learning?

Traditional programming: you write explicit rules. The computer follows them.

Machine learning: you give the computer data + a performance measure. The computer finds the rules.

**Formal definition (Tom Mitchell, 1997):**
> A computer program is said to *learn* from experience E with respect to some task T and performance measure P, if its performance at tasks in T, as measured by P, improves with experience E.

For a spam classifier:
- **T:** Classify emails as spam or not spam
- **E:** Labeled emails (spam / not spam)
- **P:** Accuracy on new, unseen emails

The key word is *generalization* — the ability to perform well on data the model has never seen.

---

## 3.2 Types of Learning

### Supervised Learning
You have labeled data — input-output pairs `{(x₁, y₁), ..., (xₙ, yₙ)}`. The goal is to learn a function `f: X → Y`.

- **Regression:** y is continuous (e.g., predicting house prices)
- **Classification:** y is categorical (e.g., spam/not-spam, digit 0–9)

### Unsupervised Learning
You have unlabeled data `{x₁, ..., xₙ}`. The goal is to find structure.

- **Clustering:** Group similar data points (K-Means, DBSCAN)
- **Dimensionality reduction:** Compress data to fewer dimensions (PCA, t-SNE)
- **Density estimation:** Learn the distribution `p(x)` (GMMs, VAEs)
- **Anomaly detection:** Find unusual points

### Semi-Supervised Learning
Large amounts of unlabeled data + small amount of labeled data. Common in practice — labeling is expensive.

### Reinforcement Learning
An agent takes actions in an environment, receives rewards, and learns a policy to maximize cumulative reward. No fixed dataset — data is generated by the agent's interactions.

### Self-Supervised Learning
A form of unsupervised learning where labels are generated from the data itself (e.g., predict the next word in a sentence). Foundation of modern LLMs.

## 3.3 The Learning Problem

### Formal Setup

- **Input space** X (e.g., ℝⁿ for tabular data)
- **Output space** Y (e.g., ℝ for regression, {0,1,...,K} for classification)
- **True distribution** P(X, Y) — unknown
- **Training set** `D = {(xᵢ, yᵢ)}ᵢ₌₁ⁿ` — i.i.d. samples from P
- **Hypothesis class** H — set of functions `f: X → Y` we consider
- **Loss function** `L(ŷ, y)` — measure of prediction error

**Goal:** Find `f ∈ H` that minimizes *expected* loss (risk):
```
R(f) = E_{(x,y)~P}[L(f(x), y)]
```

We can't minimize R(f) directly (we don't know P). Instead we minimize *empirical risk*:
```
R̂(f) = (1/n) Σᵢ L(f(xᵢ), yᵢ)
```

This is called **Empirical Risk Minimization (ERM)** — the foundation of supervised learning.

### Common Loss Functions

**Regression:**
```
MSE: L(ŷ, y) = (ŷ - y)²
MAE: L(ŷ, y) = |ŷ - y|
Huber: blend of MSE and MAE, robust to outliers
```

**Classification:**
```
0-1 loss:      L(ŷ, y) = 1[ŷ ≠ y]  (not differentiable)
Cross-entropy: L(p, y) = -y log(p) - (1-y) log(1-p)  (binary)
Cross-entropy: L(p, y) = -Σₖ 1[y=k] log(pₖ)  (multiclass)
```

Why cross-entropy for classification? It is the negative log-likelihood of the predicted probabilities under the true labels. Minimizing cross-entropy = maximizing likelihood.

## 3.4 Bias-Variance Tradeoff

This is the single most important concept for understanding model behavior.

For regression with MSE loss, the expected error can be decomposed:
```
E[(f(x) - y)²] = Bias²[f(x)] + Variance[f(x)] + Irreducible Noise
```

**Bias:** How far off is the model's average prediction from the truth?
- High bias = model is too simple (underfitting)
- Example: fitting a straight line to quadratic data

**Variance:** How much does the model's prediction change with different training sets?
- High variance = model is too sensitive to training data (overfitting)
- Example: a high-degree polynomial that fits every training point perfectly

**Irreducible noise:** Inherent randomness in the data. Cannot be reduced by any model.

```
Simple model  →  High Bias, Low Variance  →  Underfits
Complex model →  Low Bias, High Variance  →  Overfits
```

**The tradeoff:** Making a model more complex reduces bias but increases variance. The optimal complexity minimizes total error (the U-shaped test error curve).

### Visual Intuition

Imagine you're throwing darts at a target. You throw 10 sets of darts, each set trained on different data.

```
Low Bias, Low Variance:   All darts clustered at the bullseye  ✓
Low Bias, High Variance:  Darts scattered around the bullseye
High Bias, Low Variance:  Darts clustered away from center
High Bias, High Variance: Darts scattered away from center    ✗
```

## 3.5 Overfitting and Underfitting

**Overfitting:** Model performs well on training data but poorly on new data. The model has "memorized" the training data rather than learning the underlying pattern.

Signs:
- Train error << Test error
- Model is very complex (many parameters relative to data)

**Underfitting:** Model performs poorly on both training and test data. The model is too simple to capture the underlying pattern.

Signs:
- Both train and test error are high
- Model is too simple (too few parameters, too constrained)

**How to fix overfitting:**
1. Get more data (best solution when possible)
2. Reduce model complexity (fewer parameters, shallower tree)
3. Regularization (add a penalty for complexity)
4. Dropout (neural networks)
5. Early stopping (stop before the model fully memorizes)
6. Data augmentation (create more training data artificially)
7. Ensemble methods (average out the variance)

**How to fix underfitting:**
1. Use a more complex model
2. Add more features
3. Reduce regularization
4. Train longer (for neural networks)

## 3.6 Train, Validation, and Test Sets

### The Three-Way Split

```
Full Dataset
    │
    ├── Training Set (60–70%)    — used to fit the model
    ├── Validation Set (10–20%)  — used to tune hyperparameters
    └── Test Set (10–20%)        — touched ONLY at the very end
```

**Why three sets?**

If you tune hyperparameters on the test set, you are overfitting to the test set. The test set must represent unseen data. Use the validation set for all model selection decisions. Use the test set only once to report final performance.

### Data Leakage

Leakage occurs when information from the test set (or future) contaminates the training process. It causes artificially inflated metrics that don't reflect real-world performance.

Common sources:
- Scaling using statistics from the full dataset (including test)
- Using future information to predict the past (in time series)
- Including the target variable derivatively in the features

### Cross-Validation (CV)

When data is limited, a single train/val split is wasteful. k-Fold CV is better:

```
Fold 1: [TRAIN][TRAIN][TRAIN][TRAIN][VAL]
Fold 2: [TRAIN][TRAIN][TRAIN][VAL][TRAIN]
Fold 3: [TRAIN][TRAIN][VAL][TRAIN][TRAIN]
Fold 4: [TRAIN][VAL][TRAIN][TRAIN][TRAIN]
Fold 5: [VAL][TRAIN][TRAIN][TRAIN][TRAIN]
```

Train on k-1 folds, validate on the remaining fold. Repeat k times. Average the k scores.

Benefits:
- Uses all data for both training and validation
- Lower variance estimate of generalization error
- Cost: k× more training runs

**Stratified CV:** Each fold preserves the class distribution of the full dataset. Use for imbalanced problems.

```python
from sklearn.model_selection import StratifiedKFold, cross_val_score

skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(model, X, y, cv=skf, scoring='roc_auc')
```

## 3.7 Gradient Descent

Gradient descent is the optimization algorithm underlying virtually all of ML.

**Problem:** Minimize loss `L(θ)` with respect to parameters `θ`.

**Algorithm:**
```
Initialize θ randomly
Repeat until convergence:
    g ← ∇_θ L(θ)           # compute gradient
    θ ← θ - η · g          # step in negative gradient direction
```

`η` (eta) is the **learning rate** — the most important hyperparameter.

### Variants

**Batch Gradient Descent:** Compute gradient over the entire training set. Exact gradient, slow per update, smooth convergence.
```
g = (1/n) Σᵢ ∇_θ L(f(xᵢ; θ), yᵢ)
```

**Stochastic Gradient Descent (SGD):** Compute gradient on a single randomly chosen sample. Noisy but much faster.
```
g = ∇_θ L(f(xᵢ; θ), yᵢ)   for one random i
```

**Mini-batch SGD:** Compute gradient on a small batch (typically 32–256 samples). Best of both worlds. This is what's used in practice.
```
g = (1/B) Σ_{i∈batch} ∇_θ L(f(xᵢ; θ), yᵢ)
```

### Learning Rate Effects

```
Too large η:  Loss oscillates or diverges
Too small η:  Converges very slowly
Just right:   Smooth descent to minimum
```

Learning rate is usually chosen by:
1. Grid search over {0.1, 0.01, 0.001, 0.0001}
2. Learning rate schedules (decay over time)
3. Adaptive methods (Adam, RMSProp — covered in Ch 9)

### Convergence Conditions

For convex functions, GD converges to the global minimum.
For non-convex (neural networks), GD may converge to:
- A local minimum
- A saddle point (gradient = 0 but not a minimum)
- A good-enough solution (in practice, most "local minima" of deep networks are near-globally-optimal)

### Implementation from Scratch

```python
import numpy as np

def gradient_descent(f, grad_f, theta_init, lr=0.01, n_iter=1000):
    """
    f:         loss function
    grad_f:    gradient function
    theta_init: initial parameters
    """
    theta = theta_init.copy()
    history = {'loss': [], 'theta': []}
    
    for i in range(n_iter):
        loss = f(theta)
        grad = grad_f(theta)
        theta = theta - lr * grad
        
        history['loss'].append(loss)
        history['theta'].append(theta.copy())
    
    return theta, history

# Example: minimize f(x) = x^2 + 2x + 1 = (x+1)^2
f = lambda x: x**2 + 2*x + 1
grad_f = lambda x: 2*x + 2   # analytical gradient

theta_star, hist = gradient_descent(f, grad_f, np.array([5.0]), lr=0.1, n_iter=50)
print(f"Minimum at x = {theta_star[0]:.4f}")  # should be -1.0
```

## 3.8 The No Free Lunch Theorem

"Averaged over all possible problems, no learning algorithm performs better than any other."

**Practical implication:** There is no universally best algorithm. Algorithm selection depends on:
- The structure of your problem
- How much data you have
- Computational constraints
- Prior knowledge about the domain

This is why we study many algorithms — not because each is always best, but because each makes different inductive biases.

---

## 3.9 Inductive Bias

Every learning algorithm has an **inductive bias** — an implicit assumption about the form of the true function.

| Algorithm | Inductive Bias |
|-----------|---------------|
| Linear regression | True function is linear |
| k-NN | Nearby points have similar labels |
| Decision tree | Decision boundary is axis-aligned |
| SVM | Max-margin linear classifier |
| Neural network | Smooth hierarchical representations |

Choosing a model is choosing its inductive bias. Good models have biases that match the true data-generating process.

## 3.10 The ML Workflow

Every real ML project follows roughly this pipeline:

```
1. Define the problem
   └── What are you predicting? What is the metric?
   
2. Collect and understand data
   └── EDA, missing values, distributions, correlations
   
3. Preprocess and feature engineer
   └── Encoding, scaling, imputation, feature creation
   
4. Establish a baseline
   └── Simplest possible model (mean predictor, logistic regression)
   
5. Select and train models
   └── Cross-validate, compare on validation metric
   
6. Tune hyperparameters
   └── Grid search, random search, Bayesian optimization
   
7. Evaluate on test set
   └── Only once! At the end.
   
8. Error analysis
   └── Look at the misclassified examples — what do they have in common?
   
9. Iterate
   └── More data? Better features? Different model?
   
10. Deploy
    └── Serve predictions, monitor for drift
```
